# Full-modulus predecessor delay


The two-generation certificate previously checked whether a predecessor was an

## 1. Linear predecessor progression modulo X

Let `n` be a genuine full output representative, and let `s` be its least full
label:

```text
1 <= s <= O,
2^s*n == 1 (mod X).
```

All possible incoming exact valuations are `s+qO`, and their predecessors are

```text
m_q = (2^(s+qO)*n-1)/X.
```

Put

```text
U = 2^O,
C = (U-1)/X.
```

Then exactly

```text
m_(q+1)=U*m_q+C.
```

Modulo `X`, since `U==1 (mod X)`, this becomes the arithmetic progression

```text
m_q == m_0+q*C_X (mod X),                         (1)
C_X = C mod X
    = 63726582940809041391.
```

The exact certificate also verifies

```text
gcd(C_X,X)=3.
```

## 2. Exact full-output membership

Because

```text
X=M*P,
gcd(ord_M(2),ord_P(2))=1,
```

a unit `m modulo X` lies in the subgroup generated by `2` exactly when both
conditions hold:

```text
m mod M is a power of 2 modulo M,
m^ord_P(2) == 1 (mod P).
```

Define the full predecessor delay `d_X(n)` as the least `q>=0` for which the
state in (1) satisfies these two conditions.

If `n` occurs in a cycle, its actual predecessor is itself an accelerated
output.  Therefore its actual full-order layer satisfies

```text
q >= d_X(n).                                      (2)
```

Together with the full-label occupancy budget this gives

```text
sum_i d_X(n_i) <= 6257                            (3)
```

at either of the final two sparse-window lengths.

For every survivor the verifier computes the exact progression (1) until it


## 4. Significance

The former small-modulus delay was at most `41`; the full-modulus delay reaches
`347` already below one million.  More importantly, (3) means that only a tiny
selection of positive-delay values can occur in any hypothetical cycle.

This is an exact transition condition, not a statistical assumption and not a
trajectory search.  It substantially strengthens the reciprocal-selection
dual for the hardest remaining length.
